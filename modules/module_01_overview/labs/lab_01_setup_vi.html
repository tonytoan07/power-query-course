Lab 1: Thiết lập Môi trường & Khám phá Dữ liệu

Tổng quan Lab: Thiết lập môi trường Excel cho tự động hóa dữ liệu ngân hàng và khám phá bộ dữ liệu BCDKT mẫu.
Thời gian: 2 giờ | Độ khó: Cơ bản

Yêu cầu Tiên quyết

  • Microsoft Excel 365 hoặc Excel 2019
  • Quyền quản trị để cài đặt add-in
  • File dữ liệu ngân hàng mẫu (được cung cấp)
  • Hiểu biết cơ bản về giao diện Excel

Mục tiêu Lab

Sau khi hoàn thành lab này, bạn sẽ có thể:

  • Cấu hình Excel cho Power Query và data modeling
  • Kích hoạt các add-in cần thiết cho phân tích dữ liệu ngân hàng
  • Điều hướng và hiểu cấu trúc dữ liệu BCDKT mẫu
  • Xác minh môi trường sẵn sàng cho các module nâng cao

Exercise 1: Khám phá Bộ dữ liệu Lớn

1Kiểm tra Bộ dữ liệu Giao dịch

Điều hướng đến bộ dữ liệu giao dịch lớn:

  • training-materials/data/raw/transactions/ (12 file hàng tháng)
  • Mỗi file chứa ~500,000 bản ghi
  • Tổng bộ dữ liệu: ~6 triệu giao dịch ngân hàng (500K mỗi tháng)
  • training-materials/data/raw/chart_of_accounts.csv
  • training-materials/data/raw/account_hierarchy.csv

2Mở File Hàng tháng Mẫu

  1. Mở transactions_2024_01.csv trong Excel
  2. Xem lại cấu trúc dữ liệu (500K+ dòng):
    • Date: Ngày giao dịch
    • Account_Code: Mã tài khoản ngân hàng 4 chữ số
    • Description: Mô tả giao dịch bằng tiếng Việt
    • Transaction_Type: Debit hoặc Credit
    • Amount: Số tiền giao dịch bằng VND
    • Currency: Mã tiền tệ (VND, USD, EUR)
  3. Lưu ý khối lượng dữ liệu và hiệu suất Excel

Exercise 2: Kiểm tra Hiệu suất Hệ thống

1Kiểm tra Import File Lớn

  1. Tạo workbook Excel mới
  2. Đi đến Data > Get Data > From File > From Text/CSV
  3. Chọn training-materials/data/raw/transactions/transactions_2024_01.csv (500,000 bản ghi)
  4. Nhấp Load (không phải Transform Data)
  5. Giám sát hiệu suất Excel và sử dụng bộ nhớ

Import file CSV giao dịch qua Get Data

Ghi chú Hiệu suất:
  • Việc tải có thể mất 30-60 giây
  • Excel nên xử lý 500K dòng một cách hiệu quả
  • Nếu chậm, hãy đóng các ứng dụng khác

2Xác minh Tải Dữ liệu

  1. Kiểm tra thanh trạng thái - nên hiển thị ~500,000 dòng
  2. Cuộn qua dữ liệu để xác minh tính đầy đủ
  3. Kiểm tra tất cả các cột đã tải chính xác
  4. Lưu ý hiệu suất Excel với bộ dữ liệu lớn

Xác minh dữ liệu đã tải chính xác với 500K dòng

3Lưu và Đóng

  1. Lưu workbook dưới tên "Performance_Test.xlsx"
  2. Đóng file để giải phóng bộ nhớ
  3. Điều này xác nhận hệ thống có thể xử lý toàn bộ bộ dữ liệu

Exercise 3: Xem trước Dữ liệu Dashboard

1Phân tích Dữ liệu Cơ bản

  1. Với dữ liệu giao dịch đã tải, tạo pivot table đơn giản:
    • Đi đến Insert > PivotTable
    • Kéo TransactionType vào Rows
    • Kéo Amount vào Values
    • Điều này hiển thị tổng debit và credit
  2. Bạn nên thấy:
    • DEBIT: Số tiền dương lớn (~$8.6B)
    • CREDIT: Số tiền âm lớn (~-$8.6B)
    • Grand Total: Nên bằng 0.00 (cân bằng)
  3. Xác minh kế toán đúng: Debits (dương) + Credits (âm) = 0
  4. Điều này xác nhận nguyên tắc kế toán kép

Pivot table hiển thị tổng debit và credit cân bằng

2Hiểu Mục tiêu Cuối cùng

  1. Xem lại hình ảnh dashboard mục tiêu trong training-materials/data/sample/
  2. Lưu ý các chỉ số chính chúng ta sẽ tạo:
    • Total Assets: $4.2M
    • Total Liabilities: $1.8M
    • Total Equity: $2.4M
    • Current Ratio: 1.5

3Kết nối Dữ liệu với Mục tiêu

  1. 500K bản ghi giao dịch sẽ tổng hợp thành các giá trị tóm tắt này
  2. Mã tài khoản sẽ ánh xạ với danh mục Tài sản, Nợ phải trả, Vốn chủ sở hữu
  3. So sánh kỳ sẽ hiển thị xu hướng Q1, Q2, Q3 2024
  4. Biểu đồ sẽ trực quan hóa cấu trúc tài sản và thành phần nợ phải trả
Kết quả Mong đợi: Dữ liệu mẫu được thiết kế để cân bằng, thể hiện nguyên tắc kế toán kép đúng đắn.

Exercise 4: Tải và Xác thực Kết quả

1Xác minh Cân bằng Dữ liệu

  1. Kiểm tra pivot table hiển thị giao dịch cân bằng:
    • Tổng DEBIT nên bằng tổng CREDIT (về giá trị tuyệt đối)
    • Điều này xác nhận kế toán kép đúng đắn
    • Bất kỳ mất cân bằng nào đều cho thấy vấn đề chất lượng dữ liệu
  2. Lưu ý tổng khối lượng giao dịch để tham khảo

2Lưu Công việc

  1. Lưu workbook dưới tên "Module_01_Data_Exploration.xlsx"
  2. Điều này sẽ phục vụ như baseline cho Module 2
  3. Đóng Excel để giải phóng tài nguyên hệ thống

3Xác thực Môi trường

  1. Xác nhận hệ thống đã thành công:
    • Tải bộ dữ liệu 500K+ dòng mà không có lỗi
    • Tạo phân tích pivot table
    • Xác minh cân bằng và cấu trúc dữ liệu
  2. Bạn đã sẵn sàng cho các thao tác Power Query trong Module 2
