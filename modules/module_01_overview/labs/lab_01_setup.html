Lab 1: Environment Setup & Data Exploration

Lab Overview: Set up your Excel environment for banking data automation and explore sample BCDKT datasets.
Duration: 2 hours | Difficulty: Beginner

Prerequisites

  • Microsoft Excel 365 or Excel 2019
  • Administrative rights to install add-ins
  • Sample banking data files (provided)
  • Basic understanding of Excel interface

Lab Objectives

By the end of this lab, you will be able to:

  • Configure Excel for Power Query and data modeling
  • Enable necessary add-ins for banking data analysis
  • Navigate and understand sample BCDKT data structure
  • Verify your environment is ready for advanced modules

Exercise 1: Large Dataset Exploration

1Examine the Transaction Dataset

Navigate to the large transaction dataset:

  • training-materials/data/raw/transactions/ (12 monthly files)
  • Each file contains ~500,000 records
  • Total dataset: ~6 million banking transactions (500K per month)
  • training-materials/data/raw/chart_of_accounts.csv
  • training-materials/data/raw/account_hierarchy.csv

2Open Sample Monthly File

  1. Open transactions_2024_01.csv in Excel
  2. Review the data structure (500K+ rows):
    • Date: Transaction date
    • Account_Code: 4-digit banking account code
    • Description: Transaction description in Vietnamese
    • Transaction_Type: Debit or Credit
    • Amount: Transaction amount in VND
    • Currency: Currency code (VND, USD, EUR)
    • TransactionID: Unique identifier
    • Date: Transaction date
    • AccountCode: Banking account code
    • TransactionType: DEPOSIT, WITHDRAWAL, TRANSFER, etc.
    • Currency: VND, USD, EUR
    • Amount: Transaction amount
    • Branch: Branch code (BR001-BR020)
  3. Notice the realistic data volumes (500K+ rows)
  4. Check account code formats (4-digit codes)
  5. Verify date ranges (full 2024 year)

3Examine Account Information

  1. Open training-materials/data/raw/chart_of_accounts.csv
  2. Review the structure:
    • Account_Code: Links to transaction data
    • Account_Name: Business-friendly names
    • Account_Type: Asset, Liability, Equity
    • BCDKT_Category: Vietnamese balance sheet classification
  3. This will map transactions to balance sheet categories

  3. Select training-materials/data/raw/transactions/transactions_2024_01.csv (500,000 records)
  4. Click Load (not Transform Data)
  5. Monitor Excel performance and memory usage

Import transaction CSV file via Get Data

  2. Scroll through data to verify completeness
  3. Check that all columns loaded correctly

Verify data loaded correctly with 500K rows

Exercise 3: Dashboard Data Preview

1Basic Data Analysis

  1. With the transaction data loaded, create a simple pivot table:
    • Go to Insert > PivotTable
    • Drag TransactionType to Rows
    • Drag Amount to Values
    • This shows debit and credit totals
  2. You should see:
    • DEBIT: Large positive amount (~$8.6B)
    • CREDIT: Large negative amount (~-$8.6B)
    • Grand Total: Should be 0.00 (balanced)
  3. Verify proper accounting: Debits (positive) + Credits (negative) = 0
  4. This confirms double-entry bookkeeping principles

Pivot table showing balanced debit and credit totals

2Understand Final Goal

  1. Review the target dashboard image in training-materials/data/sample/
  2. Note the key metrics we'll create:
    • Total Assets
    • Total Liabilities
    • Total Equity
    • Current Ratio

3Connect Data to Goal

  1. The 500K transaction records will aggregate to these summary values
  2. Account codes will map to Assets, Liabilities, Equity categories
  3. Period comparison will show monthly 2024 trends
  4. Charts will visualize asset structure and liability mix
Expected Result: The sample data is designed to be balanced, demonstrating proper double-entry bookkeeping principles.

Exercise 4: Load and Validate Results

1Verify Data Balance

  1. Check that your pivot table shows balanced transactions:
    • DEBIT total should equal CREDIT total (in absolute values)
    • This confirms proper double-entry bookkeeping
    • Any imbalance indicates data quality issues
  2. Note the total transaction volume for reference

2Save Your Work

  1. Save the workbook as "Module_01_Data_Exploration.xlsx"
  2. This will serve as your baseline for Module 2
  3. Close Excel to free system resources

3Environment Validation

  1. Confirm your system successfully:
    • Loaded 500K+ row dataset without errors
    • Created pivot table analysis
    • Verified data balance and structure
  2. You're now ready for Power Query operations in Module 2
